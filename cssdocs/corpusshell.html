The style for the corpus_shell HTML5 app

Setting html and body to 100% is the HTML5 way of using the whole window. Other parts of the page have to be configured to expand as well. Failure to do this results in the gray (#CCCCCC) background being visible. The generic table setting is used by the "window captions". Note/Todo: The example below looks broken and it actually is. The problem with the idea of organizing the title bar as a table is that this violates the specs. A paragraph as used here as a container in principle cannot hold other block level elements such as a table so eg. Firefox interprets the paragraph as two paragraph one before and one after the table. Blind tables aren't used that much anymore in web sites in 2013 so this should be changed in the long run.

Example:

html {
	height: 100%;
	width: 100%;
	background-color: #CCCCCC;
	font-family: verdana, helvetica, serif;
	font-size: 11px;
}
body {
	height: 100%;
	width: 100%;
	margin: 0px;
	padding: 0px;
}

table {
	font-size: 11px;
}

div p.ui-widget-header {
	cursor: move;
	border: 1px solid #aaaaaa;
	background: #cccccc url(base/images/ui-bg_highlight-soft_75_cccccc_1x100.png) 50% 50% repeat-x;
	color: #222222;
	font-weight: bold;
}

Side bar: The initial user interface

The sidebar is a table with a single column and two rows. All ui elements are placed in the first row. The buttons are styled a elements. Their class is "searchresult" (for whatever reason). There is a default margin for h3 elements (14px). As we don't use h3 in the usual way this has to be reset to 0. Note/Todo: This use of of a blind table is discouraged in web designs in 2013. This should be changed in the long run.
